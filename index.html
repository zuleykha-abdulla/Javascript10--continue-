AJAX

Ajax (Asynchronous JavaScript And XML) veb səhifə daxilində səhifə yenilənmədən məlumatların serverdən götürülüb səhifəyə yerləşdirilməsi və məlumatları serverə göndərmək üçün istifadə olunur. Php kimi dillərdə əməliyyatlar baş verərkən mütləq səhifə yenilənməlidir. Məsələn, istifadəçi qeydiyyatdan keçirsə qeydiyyat prosesinin tamamlanması üçün səhifə yenilənməlidir. Ancaq Php ilə bərabər Javascript daxilindəki Ajax mövzusunu istifadə edərək səhifəni yeniləmədən əməliyyatlar apara bilərik.


Ajax nədir və necə işləyir?

Ajax, izahı ilə Asynchronous JavaScript və XML, Türk dilində XLM qısaltması verilən addır. O, serverlərə müraciətlərin arxa planda işlənməsini təmin edir və saytların asinxron olmadığı halda çiçəklənməsinə kömək edir. Ajax nədir və necə işləyir sualını bu şəkildə qısaca izah etmək olar. Ajax haqqında bütün ətraflı sual və cavablara qalan məzmundan daxil ola bilərsiniz. Ağlınızda olan hər hansı sual işarəsindən qaçına bilərsiniz. Ən əsas terminlərə görə Ajax Javascript tərəfindən tanınan kodlaşdırma dili adlandırıla bilər. Bütün bu xüsusiyyətlərlə yanaşı, Ajax sayəsində veb saytların dinamik məzmunu idarə olunur və dinamik struktur quraraq istifadəçi qarşılıqlı əlaqəsi təmin edilir. Adından da göründüyü kimi Xtensible Markup Language HTML və XML-in oxşarlığıdır. Bu dil tez-tez istifadə olunur və hər kəs tərəfindən bilinir. Bu vəziyyəti qısaca olaraq aşağıdakı kimi izah etmək olar; HTML məlumatları təmsil etmək üçün nəzərdə tutulub, XML isə məlumatları əhatə edir. JavaScript, Ajax və ya XML olmasından asılı olmayaraq, onların hamısı asinxron işləyir. Bu səbəbdən Ajax-ın istifadə olunduğu hər bir saytda səhifəni yeniləmədən məlumat göndərmək və geri almaq mümkündür.

Ajaxdan istifadənin praktiki olması üçün tövsiyələr

Ajaxdan istifadənin praktiki olması üçün tövsiyələr Google istifadəsinin ilk illərində, istifadəçilərinə yeni təkliflər təqdim edərkən, hər bir təklif bir səhifə təravəti tələb edirdi. Mövzu ilə bağlı araşdırma apararkən Google tərəfindən verilən açar sözlər bu təkliflərə nümunə ola bilər. Ajax isə hər bir təklifdə səhifəni yeniləmək ehtiyacının qarşısını alır və daha yaxşı istifadəçi təcrübəsi təqdim edir. Ajax 90-cı illərdən bəri ətrafda olub. Lakin Google 2004-cü ildə ilk dəfə Google Maps-da Ajax-dan istifadə etməyə başladı və sonra daha çox tətbiqlərə yayıldı. Bu gün daha yaxşı nəticələr əldə etmək və istifadəçi təcrübəsinin zirvədə olmasını təmin etmək məqsədilə saytlarda geniş istifadə olunur. Ajaxın istifadəsi ilə yaranan praktik cəhətlər aşağıdakılardır: Skorinq və səsvermə sistemlərində istifadə olunur: Siz nə vaxtsa alış-veriş və ya qeydiyyatdan online? Əgər cavabınız bəlidirsə, demək olar ki, Ajax bu məqamda sizə kömək edə bilər. Ajax internet vasitəsilə təklif olunan xidmətlərin səsvermə və skorinqə icazə verir. Ajax-ın istifadəsi sayəsində siz səsinizi verəndən sonra səhifə bağlanmadan davam edir. Söhbət otaqlarında istifadə olunur: Söhbət otaqlarında onlayn zəng mərkəzlərinin fəaliyyətində Ajax-dan istifadənin də təmin edildiyini demək olar. Ajax; Söhbətin səhifə təravəti olmadan davam etməsi, axının narahat olmaması və söhbətin eyni ekranda davam etməsi təmin edir. Bu yolla mesaj aldıqda və ya mesaj göndərdikdə söhbət səhifəni yeniləmədən davam edir. Sosial media gündəm bildirişlərində istifadə olunur: Ajax sosial media tətbiqlərini yeniləmək və anında bildirişlər vermək üçün istifadə olunur. Hər dəfə bildiriş alınır və ya paylaşılır, səhifə yenilənmədən yenilənmə təmin edilir. Bütün bunlar Ajax tərəfindən mümkün olur. Ajax nədir və nə edir? Sualı bu cür izah etmək mümkündür.

Ajax necə işləyir?

Ajax haqqında tez-tez verilən digər bir sual Ajax necə işləyir? Ajax təkcə inkişaf etmiş texnologiya və proqramlaşdırma dili deyil. Unutmaq olmaz ki, Ajax bir veb inkişaf texnikasıdır və onun içində bir çox sistem var. Onun daxil olduğu sistemləri aşağıdakı kimi sıralana bilər: Təqdimat üçün CSS daxildir. Evdə yerli dil dəstəyi HTML və XHTML in-house. Buraya "Document Object Model" adlı DOM və "Document Object Model" daxildir ki, görüntü məlumatları dinamik şəkildə təmin edilsin və onunla qarşılıqlı əlaqədə olsun. Məlumat mübadiləsi üçün XML və manipulyasiya üçün XSLT daxildir. Burada asinxron rabitəni təmin etmək məqsədilə XMLHttpRequest obyektləri yerləşir. Sonuncu hissədə bütün texnologiyaları bir araya gətirmək üçün JavaScript-dən istifadə edir. Bu əməkdaşlıqla çox yüksək keyfiyyətli nəticə verir. Ajax-dan necə istifadə etmək sualına cavab bir az belədir. Çünki Ajaxın bir cəhəti yoxdur, lakin bir-biri ilə əlaqəli olan bir çox xüsusiyyət və aspektlər vardır.

Klassik Model və Ajax Modeli arasındakı fərqlər hansılardır?

Klassik model ilə Ajax modeli arasındakı fərqlər xeyli çoxdur. İlk növbədə Classic Model incələndikdə bu modelin internet brauzerindən serverə http sifarişini göndərdiyi müşahidə olunur. Ajax modeli isə brauzerə xas JavaScript zəngi yaradır. Sonra bu çağırış XMLHttoRequest-in aktivləşdirilməsi ilə dəstəklənir. Classic Model sifarişi serverdən əldə edir və əməliyyat yerinə yetiriləndən sonra məlumatları götürə bilər. Ajax isə əksinə olaraq arxa planda http istəyi yaradır. Classic Model əldə etdiyi məlumatları veb-brauzerə göndərdiyi halda, Ajax heç bir məlumat almaqdan bezmir. Quraşdırılmış sistem sayəsində server məlumatları əldə edir və brauzerə göndərir. Sonuncu mərhələdə, Classic Modeldə brauzer məlumatları alır və səhifəni yeniləyir. Ajax isə heç bir səhifə təravətləndirmə yerinə yetirmir, bütün əməliyyatları səhifəni yeniləmədən etmək olar. İki model arasında ən aydın fərq Klassik Modelin istifadəçi təcrübəsinin daha aşağı olmasıdır. Çünki proses bitənə qədər istifadəçilərin gözləməkdən başqa yolu qalmır. Əgər Ajax istifadə olunursa, gözləmə müddəti yoxdur, serverlər və digər texniki hissələr mütəşəkkil qaydada işləyir. Gözlənilən səhifə dərhal görünür və heç bir səhifə təravəti tələb olunmur. Üstünlüklərə nəzər salsaq, Ajaxdan istifadə etməyin bir çox faydası olduğunu qeyd etmək mümkündür. Burada Ajax haqqında maraqlı olan və bilməli olduğunuz şeylər var. Ajaxın istifadəsi ilə hər şeyi daha qısa zamanda etmək çox asandır.
